### ✅ 如何处理 Kafka 消费者遭遇的 Offset 问题？

**扩展与回答：**

Kafka 的 Offset 问题一般包括以下几种情况：

1. **消费者重复消费（重复 offset）**：可通过配置 `enable.auto.commit=false`，在处理完消息后手动提交 offset 保证“至少一次”语义。
2. **消费者漏消费（offset 丢失）**：使用消费者组的 `group.id` 配置确保 offset 被 Kafka 自动管理或保存在外部（如数据库）。
3. **offset 被清除（过期）**：Kafka 默认只保留一定时间的 offset 数据（`offset.retention.minutes`），如果消费太慢或中断太久会导致 offset 被删除。此时消费者需要指定 `auto.offset.reset=earliest` 或 `latest` 来决定是从头读还是跳过。

**建议**：

- 使用 Kafka 自带的消费组管理机制。
- 控制 offset 提交策略：自动提交 vs 手动提交。
- 设置合理的 `auto.offset.reset` 策略和 offset 存储方式（Kafka / Zookeeper / 外部系统）。

### ✅ 当在一个分支上工作，但突然需要切换到另一个分支…

**扩展与回答：**

如果你当前工作目录有未提交的更改，直接切换分支会失败或产生冲突。

**解决办法**：

1. **使用 Git Stash 暂存修改**：

   ```bash
   git stash
   git checkout other-branch
   ```

2. **修改完切回原分支并恢复修改**：

   ```bash
   git checkout original-branch
   git stash pop
   ```

也可用 `git stash -u` 暂存包括未追踪的文件。

### ✅ 取消暂停容器中所有的进程的命令：

**扩展与回答：**

在 Linux 容器中可以使用以下命令取消暂停所有进程：

```bash
kill -CONT -1
```

说明：

- `-CONT` 信号用于恢复被 `SIGSTOP` 暂停的进程。
- `-1` 表示对容器内所有进程发送该信号。

也可使用 `docker container unpause <container_id>` 取消暂停整个容器。

**扩展与回答：**

```sql
CREATE TABLE student_scores (
    id INT PRIMARY KEY AUTO_INCREMENT,
    student_id INT NOT NULL,
    course_id INT NOT NULL,
    score DECIMAL(5,2) NOT NULL,
    exam_date DATE,
```

### ✅ 限制 goroutine 的并发执行数量：

**扩展与回答**：

可以使用带缓冲的 channel 作为并发控制器：

```go
semaphore := make(chan struct{}, 10) // 限制最大10个并发

for _, task := range tasks {
    semaphore <- struct{}{}
    go func(task Task) {
        defer func() { <-semaphore }()
        doWork(task)
    }(task)
}
```
